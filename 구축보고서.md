# LostOnCampus 구축 보고서

> **프로젝트명:** LostOnCampus  
> **주제:** 가천대 캠퍼스 분실물 찾기 게시판  
> **작성일:** 2026-06-30  
> **기술 스택:** Naver Cloud Platform, Ubuntu 24.04, Apache 2.4, PHP 8, Cloud DB for MySQL 8.4  
> **평가 기준:** Apache · MySQL + LiteLLM 웹사이트 구축 프로젝트 100점 평가표

---

## 1. 프로젝트 개요

LostOnCampus는 캠퍼스 내 분실물과 습득물 정보를 등록, 조회, 검색할 수 있는 웹 기반 게시판이다. 사용자는 분실물/습득물 게시글을 작성하고, 목록과 상세 화면에서 내용을 확인하며, 댓글과 문의/제보 기능을 사용할 수 있다. 게시글 수정과 삭제는 작성 시 입력한 비밀번호로 처리한다.

본 프로젝트는 3일 안에 시연 가능한 MVP를 목표로 하였고, 평가 기준에 맞춰 다음 기능을 구현했다.

| 구분 | 구현 내용 |
|---|---|
| 메인 화면 | 서비스 안내, 검색, 최근 게시글 표시 |
| 게시판 기능 | 등록, 목록, 상세, 수정, 삭제 |
| 이미지 업로드 | 게시글 등록/수정 시 이미지 첨부 |
| 댓글 기능 | 게시글 상세 화면에서 댓글 등록 및 조회 |
| 검색 기능 | 제목, 내용, 장소 기준 키워드 검색 |
| 문의/제보 | 이름, 연락처, 메시지 저장 |
| 오류 처리 | 빈 입력, 잘못된 ID, 비밀번호 오류, DB 오류 안내 |
| 보안 | `.env` 기반 DB 정보 분리, Prepared Statement, 비밀번호 해시 저장 |

## 3. 구축 과정

### 3.1 Apache + PHP 환경 구축

Ubuntu 서버에서 Apache와 PHP, MySQL 클라이언트를 설치하고 Apache 서비스를 실행했다.

```bash
sudo apt update
sudo apt install -y apache2 php libapache2-mod-php php-mysql php-cli php-mbstring php-curl php-xml mysql-client
sudo systemctl enable --now apache2
sudo systemctl restart apache2
```

확인 명령:

```bash
systemctl status apache2
php -v
php -m | grep -i mysql
php -m | grep -i pdo
```

### 3.2 MySQL 접속 및 DB 생성

Cloud DB for MySQL은 Private 도메인으로 접속하였다.

```bash
mysql -h db-47u9u6.vpc-cdb.ntruss.com -P 3306 -u DB계정명 -p
```

프로젝트 DB:

```sql
CREATE DATABASE lost_on_campus
DEFAULT CHARACTER SET utf8mb4
COLLATE utf8mb4_unicode_ci;
```

### 3.3 테이블 생성 및 마이그레이션

초기 테이블 생성은 `sql/schema.sql`로 관리한다.

```sql
SOURCE /root/web_project/sql/schema.sql;
```

기존 테이블에 이미지 업로드와 댓글 기능을 추가하기 위해 다음 마이그레이션을 적용했다.

```sql
SOURCE /root/web_project/sql/migration_20260630_comments_images.sql;
```

### 3.4 민감정보 분리

DB 접속 정보는 Git에 커밋하지 않고 웹 루트 바깥의 `/var/www/.env` 파일에 저장했다.

```bash
sudo cp /root/web_project/.env.example /var/www/.env
sudo nano /var/www/.env
sudo chown root:www-data /var/www/.env
sudo chmod 640 /var/www/.env
```

`config/db.php`는 `.env` 파일 또는 환경변수에서 `LOC_DB_HOST`, `LOC_DB_PORT`, `LOC_DB_NAME`, `LOC_DB_USER`, `LOC_DB_PASS` 값을 읽는다.

### 3.5 웹 루트 배포

애플리케이션 파일만 `/var/www/html`에 배포했다.

```bash
sudo rsync -a /root/web_project/index.php /root/web_project/config /root/web_project/includes /root/web_project/css /root/web_project/js /root/web_project/pages /root/web_project/api /root/web_project/uploads /var/www/html/
sudo chown -R www-data:www-data /var/www/html
sudo find /var/www/html -type d -exec chmod 755 {} \;
sudo find /var/www/html -type f -exec chmod 644 {} \;
sudo chmod 775 /var/www/html/uploads /var/www/html/uploads/items
sudo systemctl restart apache2
```

## 4. 데이터베이스 설계

### 4.1 `lost_items`

| 컬럼 | 타입 | 설명 |
|---|---|---|
| `id` | `INT` | 게시글 고유 번호 |
| `item_type` | `ENUM('lost','found')` | 분실물/습득물 구분 |
| `title` | `VARCHAR(200)` | 제목 |
| `location` | `VARCHAR(100)` | 분실/습득 장소 |
| `content` | `TEXT` | 상세 내용 |
| `contact` | `VARCHAR(100)` | 연락처 |
| `image_path` | `VARCHAR(255)` | 업로드 이미지 경로 |
| `edit_password` | `VARCHAR(255)` | 수정/삭제 비밀번호 해시 |
| `created_at` | `DATETIME` | 등록일 |
| `updated_at` | `DATETIME` | 수정일 |

### 4.2 `item_comments`

| 컬럼 | 타입 | 설명 |
|---|---|---|
| `id` | `INT` | 댓글 고유 번호 |
| `item_id` | `INT` | 연결된 게시글 ID |

| `content` | `TEXT` | 댓글 내용 |
| `created_at` | `DATETIME` | 등록일 |

## 5. 구현 기능 상세

### 5.1 게시글 기능

| 기능 | 구현 파일 | 처리 내용 |
|---|---|---|
| 목록 조회 | `pages/item_list.php` | 최신순 게시글 조회 |
| 상세 조회 | `pages/item_view.php` | 게시글 상세, 이미지, 댓글 표시 |
| 등록 화면 | `pages/item_write.php` | 게시글 입력 폼 |
| 수정 화면 | `pages/item_edit.php` | 기존 게시글 수정 폼 |
| 등록/수정/삭제 처리 | `api/item_action.php` | DB 저장, 수정, 삭제, 비밀번호 검증 |

### 5.2 이미지 업로드

- 지원 형식: JPG, PNG, WEBP, GIF
- 최대 크기: 3MB
- 저장 위치: `uploads/items/`
- DB 저장: `lost_items.image_path`
- 보안 처리:
  - MIME 타입 검사
  - 랜덤 파일명 생성
  - `uploads/.htaccess`로 PHP 실행 차단
  - 게시글 삭제 시 첨부 이미지 삭제

### 5.3 댓글 기능

| 항목 | 내용 |
|---|---|
| 화면 | `pages/item_view.php` |
| 처리 | `api/comment_action.php` |
| DB | `item_comments` |
| 검증 | 작성자 이름, 댓글 내용 필수 입력 |
| 표시 | 게시글 상세 화면 하단 댓글 목록 |

### 5.4 검색 기능

`pages/search.php`에서 제목, 내용, 장소를 대상으로 `LIKE` 검색을 수행한다.

검색 대상 컬럼:

- `title`
- `content`
- `location`

### 5.5 문의/제보 기능

`pages/inquiry.php`에서 이름, 연락처, 메시지를 입력받고 `api/inquiry_action.php`에서 `inquiries` 테이블에 저장한다.

## 8. LiteLLM 활용 기록 요약

본 프로젝트는 LiteLLM을 개발 파트너로 활용하여 요구사항 정의, 코드 생성, 오류 해결, 기능 개선을 반복했다.

| 단계 | 활용 내용 | 산출물 |
|---|---|---|
| 요구사항 정의 | 평가 기준과 주제 후보를 바탕으로 MVP 범위 정리 | `기획문서.md` |
| 기능명세 작성 | 화면, 기능 ID, DB 구조, 테스트 기준 정리 | `기능명세서.md` |
| 코드 생성 | PHP, HTML, CSS, SQL 기본 구조 작성 | `index.php`, `pages/`, `api/`, `sql/` |
| 오류 해결 | DB 접속 문제, 권한 문제, Apache 접속 문제 해결 절차 정리 | `실행가이드.md` |
| 기능 개선 | `.env` 분리, 댓글, 이미지 업로드, 오류 처리 보강 | 소스 코드 및 마이그레이션 |

- DB 비밀번호를 코드에서 제거하고 `.env`로 분리
- 이미지 업로드 시 파일 형식, 용량, 저장 경로 검증 추가
- 댓글 기능을 별도 API와 테이블로 분리
- 오류 메시지를 사용자 친화적으로 변경
- 업로드 폴더에서 PHP 실행을 차단
- 평가표 기준에 맞춰 문서와 테스트 항목 보강
