# LostOnCampus 실행 가이드 (Docker Compose 기반)

이 문서는 LostOnCampus 웹 애플리케이션을 Docker Compose를 사용하여 배포하고 실행하는 방법을 안내합니다.

## 1. 개요

LostOnCampus 프로젝트는 전통적인 Apache+PHP 설치 방식 대신 Docker 컨테이너 환경에서 실행되도록 설계되었습니다. 웹 서버(Apache+PHP)는 Docker 컨테이너 내에서 실행되며, 데이터베이스는 네이버 클라우드 플랫폼의 Cloud DB for MySQL(외부)을 사용합니다. 사용자 업로드 파일 및 환경 설정(`.env`)은 호스트 서버와 컨테이너 간에 볼륨 바인딩되어 데이터의 영속성과 보안을 보장합니다.

## 2. 사전 준비 사항

### 2.1 호스트 서버 환경

-   **운영체제:** Ubuntu 24.04 LTS (또는 Docker와 Docker Compose가 지원되는 다른 Linux 배포판)
-   **Docker 설치:**
    ```bash
    sudo apt-get update
    sudo apt-get install ca-certificates curl gnupg
    sudo install -m 0755 -d /etc/apt/keyrings
    curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo gpg --dearmor -o /etc/apt/keyrings/docker.gpg
    sudo chmod a+r /etc/apt/keyrings/docker.gpg
    echo 
      "deb [arch="$(dpkg --print-architecture)" signed-by=/etc/apt/keyrings/docker.gpg] https://download.docker.com/linux/ubuntu 
      "$(. /etc/os-release && echo "$VERSION_CODENAME")" stable" | 
      sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
    sudo apt-get update
    sudo apt-get install docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
    sudo usermod -aG docker $USER # 현재 사용자를 docker 그룹에 추가 (로그아웃 후 재로그인 필요)
    ```
-   **Docker Compose 설치:** (위 `docker-compose-plugin` 설치 시 함께 설치되지만, 구버전 Docker에서는 별도 설치 필요)
    ```bash
    # 최신 버전 설치는 공식 문서 참조: https://docs.docker.com/compose/install/
    ```

### 2.2 Cloud DB for MySQL 설정

-   네이버 클라우드 플랫폼에서 Cloud DB for MySQL 인스턴스를 생성하고, 다음 정보를 확보합니다.
    -   DB Host (사설 IP 주소 또는 VPC 도메인)
    -   DB Port (기본값: 3306)
    -   DB Name (예: `lost_on_campus`)
    -   DB User
    -   DB Password
-   **DB 스키마 초기화:** `sql/schema.sql` 파일을 사용하여 데이터베이스 스키마를 초기화합니다.
    ```bash
    mysql -h <DB_HOST> -P 3306 -u <DB_USER> -p <DB_NAME> < /path/to/LostOnCampus/sql/schema.sql
    ```
    -   기존 `lost_items`, `inquiries` 테이블이 이미 있는 경우, `sql/migration_20260630_comments_images.sql` 파일을 실행하여 마이그레이션합니다.
        ```bash
        mysql -h <DB_HOST> -P 3306 -u <DB_USER> -p <DB_NAME> < /path/to/LostOnCampus/sql/migration_20260630_comments_images.sql
        ```

## 3. 배포 단계

### 3.1 프로젝트 클론

호스트 서버에 LostOnCampus 프로젝트 저장소를 클론합니다.
```bash
git clone [Your Repository URL]
cd LostOnCampus
```

### 3.2 환경 변수 설정 (`.env`)

민감한 DB 접속 정보는 `.env` 파일을 통해 컨테이너에 주입됩니다. 이 파일은 호스트 서버의 웹 루트 외부인 `/var/www/.env` 경로에 위치해야 합니다.

1.  **`.env` 파일 생성:** `/var/www/.env` 경로에 `.env` 파일을 생성하고, Cloud DB for MySQL 정보를 기입합니다.
    ```bash
    sudo nano /var/www/.env
    ```
    `.env` 파일 내용 예시:
    ```bash
    LOC_DB_HOST=db-47u9u6.vpc-cdb.ntruss.com
    LOC_DB_PORT=3306
    LOC_DB_NAME=lost_on_campus
    LOC_DB_USER=DB계정명
    LOC_DB_PASS=DB비밀번호
    ```
2.  **파일 권한 설정 (권장):** 보안을 위해 `.env` 파일의 소유권과 권한을 설정합니다.
    ```bash
    sudo chown root:www-data /var/www/.env
    sudo chmod 640 /var/www/.env
    ```

### 3.3 `uploads` 디렉터리 권한 설정

컨테이너 내부의 Apache 웹 서버(www-data 사용자)가 이미지 파일을 `uploads` 디렉터리에 저장할 수 있도록, 호스트 서버에서 해당 디렉터리에 대한 쓰기 권한을 부여합니다. 프로젝트 루트 디렉터리에서 아래 명령어를 실행하십시오.

```bash
sudo chmod -R 775 uploads
```

### 3.4 Docker Compose 실행

프로젝트 루트 디렉터리(`LostOnCampus` 폴더)에서 다음 명령어를 실행하여 애플리케이션 컨테이너를 빌드하고 백그라운드에서 실행합니다.

```bash
docker compose up -d --build
```
-   `-d`: 컨테이너를 백그라운드에서 실행합니다.
-   `--build`: `Dockerfile`이 변경되었거나 이미지가 존재하지 않을 경우 새로 빌드합니다. (첫 실행 시 필수)


## 4. 배포 검증

### 4.1 컨테이너 상태 확인

컨테이너가 정상적으로 실행 중인지 확인합니다.
```bash
docker compose ps
# 출력 예시:
# NAME                COMMAND                  SERVICE             STATUS              PORTS
# lost-on-campus      "docker-php-entrypoi…"   lost-on-campus-app  running             0.0.0.0:80->80/tcp
```

웹 브라우저를 열고 호스트 서버의 공인 IP 주소로 접속하여 애플리케이션이 정상적으로 동작하는지 확인합니다.
```text
http://<서버_공인IP>
```

## 5. 문제 해결 (Troubleshooting)

-   **컨테이너 시작 실패:** `docker compose logs -f lost-on-campus-app` 명령어를 사용하여 `lost-on-campus-app` 서비스의 로그를 확인합니다. `.env` 파일 경로 또는 내용 오류, PHP 확장 설치 문제 등이 원인일 수 있습니다.
-   **웹 페이지 접속 불가:**
    -   호스트 서버의 방화벽(예: `ufw status`)에서 80번 포트가 열려 있는지 확인합니다.
    -   `docker compose ps`로 컨테이너 포트 매핑이 정상인지 확인합니다.
-   **데이터베이스 연결 오류:**
    -   `/var/www/.env` 파일의 DB 접속 정보가 정확한지 확인합니다.
    -   Cloud DB for MySQL의 방화벽 설정(ACLG)에서 호스트 서버의 IP 주소 접근을 허용했는지 확인합니다.
    -   `docker exec -it lost-on-campus bash` 명령으로 컨테이너에 접속 후 `ping <DB_HOST>`로 DB 서버와의 네트워크 연결을 테스트해봅니다.
-   **권한 문제:** 이미지 업로드 등에서 권한 문제가 발생하면 `Dockerfile`의 `chown` 설정과 `./uploads` 볼륨 마운트 설정을 다시 확인합니다.
