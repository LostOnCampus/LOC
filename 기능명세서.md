# 가천대 캠퍼스 분실물 찾기 게시판 — 기능명세서

> **문서 버전:** v1.1  
> **작성일:** 2026-06-30  
> **확정 주제:** 가천대 캠퍼스 분실물 찾기 게시판  
> **개발 목표:** Apache + MySQL + PHP 기반으로 3일 안에 시연 가능한 MVP 웹사이트 구현

---

## 1. 서비스 개요

### 1.1 서비스 목적

가천대학교 캠퍼스 안에서 발생하는 분실물과 습득물 정보를 등록하고 검색할 수 있는 게시판형 웹사이트를 구축한다. 사용자는 분실물 또는 습득물 글을 등록하고, 목록·상세 화면에서 내용을 확인하며, 필요하면 연락처를 통해 작성자와 직접 연락할 수 있다.

### 1.2 핵심 사용자

| 사용자 | 설명 | 주요 이용 기능 |
|---|---|---|
| 일반 사용자 | 분실물 또는 습득물을 찾는 학생 | 목록 조회, 검색, 상세 조회, 문의/제보 작성 |
| 게시글 작성자 | 분실물/습득물 정보를 등록한 사용자 | 게시글 등록, 수정, 삭제 |

### 1.3 MVP 범위

| 구분 | 포함 여부 | 내용 |
|---|:---:|---|
| 메인 페이지 | 포함 | 분실물 게시판을 메인 페이지로 사용 |
| 분실물 게시판 | 포함 | 등록, 목록, 상세, 수정, 삭제 |
| 습득물 게시판 | 포함 | 등록, 목록, 상세, 수정, 삭제 |
| 댓글 | 포함 | 등록, 수정, 삭제 |
| 검색 | 포함 | 제목, 내용, 장소 기준 키워드 검색 |
| 오류 처리 | 포함 | 빈 입력, 잘못된 접근, DB 오류 안내 |
| 회원가입/로그인 | 부분 포함 | 학번을 아이디로 하여 간단히 구현 |
| 이미지 업로드 | 포함 | 분실물/습득물 이미지 업로드 |

## 4. 상세 기능 명세

### F-001. 메인 페이지 조회

| 항목 | 내용 |
|---|---|
| 목적 | 사용자가 사이트 목적과 주요 메뉴를 빠르게 이해하도록 한다. |
| 사용자 | 일반 사용자, 운영·시연자 |
| 진입 경로 | `/index.php` |
| 주요 내용 | 서비스명, 이용 안내, 게시판 이동 버튼, 문의/제보 이동 버튼, 최근 게시글 일부 |
| 처리 로직 | 최근 등록된 분실물/습득물 게시글 3~5개를 조회하여 표시한다. |
| 출력 정보 | 게시글 제목, 유형, 장소, 등록일 |
| 예외 처리 | DB 조회 실패 시 최근 게시글 영역에 안내 문구를 표시한다. |
| 완료 기준 | 공인 IP 또는 도메인 접속 시 메인 화면이 정상 표시된다. |

### F-002. 분실물/습득물 목록 조회

| 항목 | 내용 |
|---|---|
| 목적 | 등록된 분실물/습득물 게시글을 한눈에 확인한다. |
| 사용자 | 일반 사용자 |
| 진입 경로 | `/pages/item_list.php` |
| 처리 로직 | `lost_items` 테이블에서 최신순으로 게시글을 조회한다. |
| 출력 정보 | 번호, 유형, 제목, 장소, 등록일 |
| 정렬 기준 | `created_at DESC` |
| 예외 처리 | 게시글이 없으면 “등록된 게시글이 없습니다.” 문구를 표시한다. |
| 완료 기준 | DB에 저장된 게시글이 목록 화면에 표시된다. |

### F-003. 게시글 상세 조회

| 항목 | 내용 |
|---|---|
| 목적 | 특정 분실물/습득물의 상세 정보를 확인한다. |
| 사용자 | 일반 사용자 |
| 진입 경로 | `/pages/item_view.php?id={id}` |
| 입력값 | 게시글 ID |
| 처리 로직 | 전달받은 ID로 `lost_items` 테이블의 단일 게시글을 조회한다. |
| 출력 정보 | 유형, 제목, 장소, 내용, 연락처, 등록일, 수정일 |
| 제공 버튼 | 목록으로, 수정, 삭제 |
| 예외 처리 | ID가 없거나 존재하지 않는 게시글이면 오류 안내 후 목록으로 이동할 수 있게 한다. |
| 완료 기준 | 목록에서 선택한 게시글의 상세 정보가 정확히 표시된다. |

### F-004. 게시글 등록

| 항목 | 내용 |
|---|---|
| 목적 | 사용자가 분실물 또는 습득물 정보를 등록한다. |
| 사용자 | 게시글 작성자 |
| 진입 경로 | `/pages/item_write.php` |
| 처리 경로 | `/api/item_action.php?action=create` |
| 입력 항목 | 유형, 제목, 장소, 내용, 연락처, 수정·삭제 비밀번호 |
| 필수값 | 유형, 제목, 장소, 내용, 연락처, 비밀번호 |
| 처리 로직 | 입력값 검증 후 `lost_items` 테이블에 저장한다. 비밀번호는 `password_hash()`로 암호화한다. |
| 성공 처리 | 저장 후 목록 또는 상세 화면으로 이동한다. |
| 예외 처리 | 필수값 누락 시 저장하지 않고 안내 메시지를 표시한다. |
| 완료 기준 | 등록한 게시글이 MySQL에 저장되고 목록에서 조회된다. |

### F-005. 게시글 수정

| 항목 | 내용 |
|---|---|
| 목적 | 작성자가 기존 게시글 내용을 수정한다. |
| 사용자 | 게시글 작성자 |
| 진입 경로 | `/pages/item_edit.php?id={id}` |
| 처리 경로 | `/api/item_action.php?action=update` |
| 입력 항목 | 게시글 ID, 유형, 제목, 장소, 내용, 연락처, 비밀번호 |
| 처리 로직 | 게시글 ID 존재 여부를 확인하고, 입력한 비밀번호를 `password_verify()`로 검증한 뒤 수정한다. |
| 성공 처리 | 수정 완료 후 상세 화면으로 이동한다. |
| 예외 처리 | 비밀번호 불일치, 필수값 누락, 존재하지 않는 ID에 대해 안내한다. |
| 완료 기준 | 수정한 내용이 DB에 반영되고 상세 화면에서 변경 내용이 표시된다. |

### F-006. 게시글 삭제

| 항목 | 내용 |
|---|---|
| 목적 | 작성자가 등록한 게시글을 삭제한다. |

| 처리 경로 | `/api/item_action.php?action=delete` |
| 입력 항목 | 게시글 ID, 비밀번호 |
| 처리 로직 | 게시글 ID 존재 여부와 비밀번호를 확인한 뒤 해당 게시글을 삭제한다. |
| 성공 처리 | 삭제 완료 후 목록 화면으로 이동한다. |
| 예외 처리 | 비밀번호 불일치, 존재하지 않는 ID에 대해 안내한다. |
| 완료 기준 | 삭제된 게시글이 목록과 검색 결과에 더 이상 표시되지 않는다. |

### F-007. 키워드 검색

| 항목 | 내용 |
|---|---|

| 사용자 | 일반 사용자 |
| 진입 경로 | `/pages/search.php?keyword={keyword}` |

| 완료 기준 | 제목·내용·장소에 포함된 키워드로 관련 게시글이 조회된다. |

### F-010. 오류 안내

| 항목 | 내용 |
|---|---|
| 목적 | 오류 상황에서도 사용자가 다음 행동을 알 수 있도록 한다. |

| 완료 기준 | 주요 오류 상황에서 흰 화면이나 PHP 오류 원문이 노출되지 않는다. |

## 5. 데이터베이스 명세

### 5.1 `lost_items` 테이블

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|---|---|---|---|
| `id` | `INT` | PK, AUTO_INCREMENT | 게시글 고유 번호 |
| `item_type` | `ENUM('lost','found')` | NOT NULL, DEFAULT `'lost'` | 분실물/습득물 구분 |
| `title` | `VARCHAR(200)` | NOT NULL | 게시글 제목 |
| `location` | `VARCHAR(100)` | NOT NULL | 분실 또는 습득 장소 |
| `content` | `TEXT` | NOT NULL | 상세 설명 |
| `contact` | `VARCHAR(100)` | NOT NULL | 연락처 |
| `edit_password` | `VARCHAR(255)` | NOT NULL | 수정·삭제용 비밀번호 해시 |
| `created_at` | `DATETIME` | DEFAULT CURRENT_TIMESTAMP | 등록일 |
| `updated_at` | `DATETIME` | DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 수정일 |

### 5.2 `inquiries` 테이블

| 컬럼명 | 타입 | 제약 조건 | 설명 |
|---|---|---|---|
| `id` | `INT` | PK, AUTO_INCREMENT | 문의 고유 번호 |
| `name` | `VARCHAR(100)` | NOT NULL | 문의자 이름 |
| `contact` | `VARCHAR(100)` | NOT NULL | 문의자 연락처 |
| `message` | `TEXT` | NOT NULL | 문의 또는 제보 내용 |
| `created_at` | `DATETIME` | DEFAULT CURRENT_TIMESTAMP | 등록일 |

### 5.3 SQL 생성문

```sql
CREATE TABLE lost_items (
    id INT AUTO_INCREMENT PRIMARY KEY,
    item_type ENUM('lost', 'found') NOT NULL DEFAULT 'lost',
    title VARCHAR(200) NOT NULL,
    location VARCHAR(100) NOT NULL,
    content TEXT NOT NULL,
    contact VARCHAR(100) NOT NULL,
    edit_password VARCHAR(255) NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
);

CREATE TABLE inquiries (
    id INT AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL,
    contact VARCHAR(100) NOT NULL,
    message TEXT NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP
);
```

## 6. 화면별 입력·출력 명세

### 6.1 게시글 등록/수정 폼

| 항목 | 입력 방식 | 필수 | 예시 | 검증 |
|---|---|:---:|---|---|
| 유형 | 선택 버튼 또는 셀렉트 | 예 | 분실물, 습득물 | `lost`, `found` 중 하나 |
| 제목 | 텍스트 입력 | 예 | 학생증을 찾습니다 | 1~200자 |
| 장소 | 텍스트 입력 | 예 | 비전타워 1층 | 1~100자 |
| 내용 | 긴 글 입력 | 예 | 검은색 카드지갑입니다. | 빈 값 금지 |
| 연락처 | 텍스트 입력 | 예 | 010-0000-0000 | 1~100자 |
| 비밀번호 | 비밀번호 입력 | 예 | 직접 입력 | 빈 값 금지, 해시 저장 |

## 8. 비기능 요구사항

| 구분 | 요구사항 |
|---|---|
| 접속성 | Naver Cloud 공인 IP 또는 도메인으로 접속 가능해야 한다. |
| 호환성 | 최신 Chrome 브라우저에서 주요 기능이 정상 동작해야 한다. |
| 보안 | SQL Injection 방지를 위해 Prepared Statement를 사용한다. |
| 보안 | 수정·삭제 비밀번호는 평문으로 저장하지 않는다. |
| 보안 | DB 접속 정보는 `config/db.php`에 분리하고 화면에 노출하지 않는다. |
| 성능 | MVP 기준 게시글 수가 적은 상황에서 목록과 검색이 3초 이내 표시되어야 한다. |
| 유지보수 | 공통 헤더/푸터를 분리하여 화면별 중복 코드를 줄인다. |

## 9. 테스트 및 완료 기준

### 9.1 기능 테스트 체크리스트

| 번호 | 테스트 항목 | 기대 결과 |
|---:|---|---|
| 1 | 공인 IP 또는 도메인으로 접속 | 메인 페이지가 정상 표시된다. |
| 2 | 게시글 등록 | 입력값이 DB에 저장되고 목록에 표시된다. |
| 3 | 게시글 목록 조회 | 최신순으로 게시글이 표시된다. |
| 4 | 게시글 상세 조회 | 선택한 게시글의 상세 정보가 표시된다. |
| 5 | 게시글 수정 | 비밀번호 일치 시 내용이 수정된다. |
| 6 | 게시글 삭제 | 비밀번호 일치 시 목록에서 제거된다. |
| 7 | 비밀번호 오류 | 수정·삭제가 처리되지 않고 오류 안내가 표시된다. |
| 8 | 키워드 검색 | 제목, 내용, 장소에 포함된 검색 결과가 표시된다. |
| 9 | 검색 결과 없음 | 결과 없음 안내가 표시된다. |
| 10 | 문의/제보 등록 | 입력값이 `inquiries` 테이블에 저장된다. |
| 11 | 필수값 누락 | 저장되지 않고 입력 안내가 표시된다. |
| 12 | 잘못된 ID 접근 | 오류 안내 후 목록으로 이동할 수 있다. |

### 9.2 시연 완료 기준

- 메인 페이지 접속이 가능하다.
- 게시글 등록 후 MySQL 저장 결과를 확인할 수 있다.
- 저장된 게시글을 목록, 상세, 검색 화면에서 조회할 수 있다.
- 게시글 수정 또는 삭제가 정상 동작한다.
- 문의/제보 폼 입력값이 MySQL에 저장된다.
- 빈 입력, 잘못된 접근, 비밀번호 불일치 상황에서 사용자 안내가 표시된다.
- LiteLLM을 활용한 요구사항 정의, 코드 생성, 오류 수정, 기능 개선 기록을 발표 또는 보고서에서 설명할 수 있다.
